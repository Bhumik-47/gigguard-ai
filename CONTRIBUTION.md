# Contributing to GigGuard AI

Thank you for your interest in contributing to **GigGuard AI**!

GigGuard AI is an AI-powered parametric insurance platform designed to protect gig workers from income loss caused by environmental disruptions such as extreme weather events and poor air quality. Our goal is to build transparent, fair, and accessible technology that helps vulnerable workers manage financial risk.

---

# Ways to Contribute

We welcome contributions in many areas:

### Development

* Fix bugs and improve reliability.
* Add new features and integrations.
* Improve AI and risk prediction models.
* Enhance API functionality and backend services.

### Frontend & UX

* Improve user experience and accessibility.
* Build responsive and intuitive interfaces.
* Enhance dashboards and reporting tools.

### Documentation

* Improve setup guides and tutorials.
* Enhance API documentation.
* Create architecture diagrams and examples.

### Research & Analytics

* Improve environmental risk assessment logic.
* Contribute forecasting methodologies.
* Enhance transparency and explainability features.

# Getting Started

## 1. Fork the Repository

Create a fork of the repository under your GitHub account.

## 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/gigguard-ai.git
cd gigguard-ai
```

## 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

---

# Development Guidelines

Please ensure that your contributions:

* Follow the existing code style and project structure.
* Use meaningful variable and function names.
* Include comments where necessary.
* Remain focused on a single issue whenever possible.
* Maintain readability and maintainability.
* Update documentation when introducing new functionality.

---

# Pull Request Guidelines

Before submitting a Pull Request:

* Ensure your code has been tested.
* Perform a self-review.
* Reference the related issue.
* Include screenshots for UI-related changes.
* Keep the PR scope limited to the assigned issue.

### Example

```text
Fixes #123
```

---

# Ethical AI Principles

As GigGuard AI operates in the domains of AI, insurance, and worker protection, contributors should:

* Prioritize fairness and transparency.
* Avoid misleading claims about predictions or outcomes.
* Clearly communicate assumptions and limitations.
* Respect privacy and data protection standards.
* Promote responsible AI development.

---

# Reporting Issues

When opening an issue:

* Use a clear and descriptive title.
* Explain the problem or feature request thoroughly.
* Include reproduction steps if reporting a bug.
* Attach screenshots or logs when applicable.

---

# Community Standards

By participating in this project, you agree to follow the project's Code of Conduct.

Please review the `CODE_OF_CONDUCT.md` before contributing.

---

# Security

If you discover a security vulnerability:

* Do not disclose it publicly.
* Report it responsibly to the project maintainers.
* Provide sufficient information for reproduction and remediation.
